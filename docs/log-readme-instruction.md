# Инструкция по проекту

## Что это за сайт

Это не SaaS и не личный кабинет для клиентов.

Сейчас сайт работает как `tier-first` витрина и sales funnel для продуктовой услуги. Его задача не автоматизировать весь процесс, а:

- показать оффер;
- дать человеку выбрать подходящий tier;
- быстро вернуть ценовой диапазон;
- перевести человека в WhatsApp-диалог;
- сохранить лид;
- дать команде видеть, из какого канала пришел интерес.

## Что сайт делает сейчас

Сайт уже переведен с калькулятора пакетов на 3 фиксированных tier:

- `starter`
- `business`
- `growth`

Для каждого tier задается диапазон цены, а не одна жесткая цена.

Публичный flow сейчас такой:

1. человек заходит на landing;
2. выбирает tier;
3. указывает контекст;
4. фронтенд вызывает `/api/quote`;
5. сайт получает price range и готовый WhatsApp text;
6. лид отправляется через `/api/leads`;
7. дальше продажа идет вручную через WhatsApp.

## Что важно по смыслу оффера

- Обещание `48h` теперь означает `first output in 48 hours`.
- Сайт больше не должен обещать полный проект за 48 часов.
- HE и EN остаются UI-языками.
- RU и AR используются в deliverable/output контексте, а не как основные UI-локали.

## Какие основные части уже есть

### 1. Публичная pricing-воронка

Сайт показывает 3 tier вместо старого калькулятора.

Она нужна, чтобы:

- человек быстрее понял оффер;
- не выбирал из слишком сложной конфигурации;
- сразу уходил в понятный ценовой коридор.

### 2. Quote API

`/api/quote` принимает tier и контекст выбора, а возвращает:

- выбранный tier;
- price range;
- текст для WhatsApp.

Это основной слой, который превращает интерес в коммерческий ответ.

### 3. Lead capture

`/api/leads` сохраняет лида вместе с:

- контактами;
- выбранным tier;
- ценовым диапазоном;
- `acquisitionChannel`;
- source.

Это нужно, чтобы не терять входящий спрос и дальше видеть, что реально выбирают люди.

### 4. CAC admin

Во внутренней админке есть `/admin/cac`.

Там можно:

- смотреть лиды по каналам;
- вносить вручную расходы за месяц;
- видеть:
  - `leads`
  - `CPL`
  - `CAC`
  - `close rate`
  - `ROAS-like`

Это не "аналитика ради аналитики", а инструмент проверки, какой канал и какой tier вообще живой.

### 5. Legacy SEO continuity

Старые slug и старые package id не удалены полностью.

Они оставлены для:

- SEO continuity;
- старых ссылок;
- маппинга старого package flow в новую tier-first модель.

## Как с этим работать сейчас

С сайтом нужно работать как с системой проверки продаж, а не как с платформой.

Операционный сценарий такой:

1. приводишь человека на landing;
2. человек кликает tier;
3. получает ориентир по цене;
4. переходит в WhatsApp;
5. ты или команда вручную ведете диалог;
6. лид фиксируется;
7. по итогам смотришь, из какого канала пришел интерес и что конвертится.

## Что уже можно проверять по рынку

С помощью текущей версии уже можно видеть:

- какой tier чаще выбирают;
- на какой tier чаще отвечают в WhatsApp;
- из какого канала приходят лиды;
- где люди зависают;
- есть ли первые оплаченные пилоты.

Этого уже достаточно для MVP-проверки.

## Что сейчас не является приоритетом

На текущем этапе не надо первым делом строить:

- client cabinet;
- payment system;
- auto-posting;
- WhatsApp automation;
- сложную AI orchestration layer;
- white-label multi-client monster.

Пока нет реальных лидов и оплаченных пилотов, это будет лишняя инженерия.

## Что нужно делать следующим этапом

### 1. Полировка перед запуском

Нужно проверить:

- все 3 tier;
- WhatsApp CTA;
- сохранение лида;
- запись `acquisitionChannel`;
- отображение лида в CAC;
- legacy slug pages;
- HE/EN copy;
- все формулировки про `first output in 48 hours`.

### 2. Валидация продаж

Нужно не "разрабатывать дальше", а получать реальные диалоги:

- ручной outreach;
- WhatsApp-переписки;
- несколько созвонов;
- первые оплаченные тесты.

### 3. Решение по следующему слою продукта

Только после этого имеет смысл решать:

- усиливать ли `starter`;
- менять ли цены;
- убирать ли один tier;
- нужен ли кабинет;
- что именно автоматизировать.

## Итог

Сейчас у проекта уже есть рабочий MVP-скелет:

- tier-first pricing;
- quote flow;
- lead capture;
- channel tracking foundation;
- CAC admin;
- legacy SEO continuity.

Главный вопрос теперь не "что еще закодить", а:

`кто платит`, `из какого канала`, и `какой tier реально продается`.
